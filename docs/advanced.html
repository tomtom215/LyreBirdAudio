Recovery Procedures

Stream Not Starting

# Quick recovery
sudo ./lyrebird-stream-manager.sh status
sudo ./lyrebird-stream-manager.sh restart

# If that fails: force cleanup and fresh start
sudo ./lyrebird-stream-manager.sh force-stop
sudo rm -f /run/mediamtx-audio.lock /run/mediamtx-audio.pid
sudo ./lyrebird-stream-manager.sh start

Device Not Found After Reboot

# Re-scan and remap USB devices
sudo ./usb-audio-mapper.sh --rescan

# Reload udev rules (may not need reboot)
sudo udevadm control --reload-rules
sudo udevadm trigger

# Verify device symlinks exist
ls -la /dev/snd/by-usb-port/

# If symlinks missing: reboot is required
sudo reboot

Complete System Recovery

Use this procedure when the system is in an unknown or corrupted state:

# 1. Stop everything
sudo systemctl stop mediamtx-audio 2>/dev/null || true
sudo ./lyrebird-stream-manager.sh force-stop

# 2. Clean up stale files
sudo rm -f /run/mediamtx-audio.{pid,lock,restart,cleanup}
sudo rm -f /var/lib/mediamtx-ffmpeg/*.pid

# 3. Verify MediaMTX installation
sudo ./install_mediamtx.sh status
# If corrupted: sudo ./install_mediamtx.sh update

# 4. Verify device mapping
sudo ./usb-audio-mapper.sh --test

# 5. Run full diagnostics
sudo ./lyrebird-diagnostics.sh full

# 6. Fresh start
sudo ./lyrebird-stream-manager.sh start

Emergency Field Recovery (No Internet)

If in the field without internet access:

  • Use local backups — keep a USB drive with a known-good configuration tarball
  • Config backup — /etc/mediamtx/audio-devices.conf.bak (auto-created by mic-check)
  • Restore config: sudo cp /etc/mediamtx/audio-devices.conf.bak /etc/mediamtx/audio-devices.conf
  • Restart: sudo systemctl restart mediamtx-audio

Version Management

Branch Structure

Branch/TagStabilityUse
Tagged releases (e.g., v1.4.2) Stable Production deployments — recommended
main Generally stable Latest features, tested but may have minor issues
development Unstable Active development only — not for production

Check for Updates

# Interactive menu
./lyrebird-updater.sh

# Command-line status
./lyrebird-updater.sh --status

# List available versions/tags
./lyrebird-updater.sh --list

Upgrade to Latest

# Via updater (recommended — handles service coordination)
./lyrebird-updater.sh
# Select option 2: Upgrade to Latest Version

# Or manually (not recommended)
git fetch origin
git checkout $(git describe --tags --abbrev=0 origin)
chmod +x *.sh

Switch to Specific Version

# Via updater
./lyrebird-updater.sh
# Select option 3: Switch to Specific Version
# Enter: v1.4.0

# Or manually
git fetch --tags
git checkout v1.4.0
chmod +x *.sh

Rollback

# Quick rollback via updater
./lyrebird-updater.sh --rollback

# Rollback to specific version
./lyrebird-updater.sh --rollback v1.3.0

# Manual rollback via git
git reflog          # Find previous good state
git checkout HEAD~1 # Or specific commit
chmod +x *.sh

Post-Update Migrations

The updater automatically runs migrations for breaking changes (e.g., script renames). To run manually:

sudo ./lyrebird-updater.sh --migrate

Migration history is tracked in /var/lib/lyrebird/migrations/. Migrations are idempotent — safe to run multiple times.

ℹ️
Script Rename Migration (v1.4.3+)

mediamtx-stream-manager.sh was renamed to lyrebird-stream-manager.sh. Running lyrebird-updater.sh automatically updates systemd service files, cron jobs, and log paths. The orchestrator will warn you at startup if old names are detected.

API Integration

The MediaMTX REST API (port 9997) enables programmatic stream control:

# List all configured paths/streams
curl -s http://localhost:9997/v3/paths/list | python3 -m json.tool

# Get details for a specific stream
curl -s http://localhost:9997/v3/paths/get/Device_1

# Global configuration
curl -s http://localhost:9997/v3/config/global/get

Custom Automation Example

#!/bin/bash
# Auto-restart on stream failure

while true; do
    if ! ./lyrebird-stream-manager.sh status | grep -q "running"; then
        echo "Streams down, restarting..."
        sudo ./lyrebird-stream-manager.sh restart
        # Send webhook alert
        source ./lyrebird-alerts.sh
        alert_stream_down "all" "Auto-restart triggered"
    fi
    sleep 30
done

Backup & Restore

Create Backup

# Backup all configuration
sudo tar -czf lyrebird-backup-$(date +%Y%m%d).tar.gz \
  /etc/mediamtx/ \
  /etc/udev/rules.d/99-usb-soundcards.rules \
  /etc/lyrebird/ \
  /var/lib/mediamtx/ 2>/dev/null

echo "Backup created: lyrebird-backup-$(date +%Y%m%d).tar.gz"

Restore from Backup

# Restore configuration
sudo tar -xzf lyrebird-backup-20250101.tar.gz -C /

# Reload udev rules
sudo udevadm control --reload-rules
sudo udevadm trigger

# Reload systemd
sudo systemctl daemon-reload

# Restart services
sudo systemctl restart mediamtx
sudo ./lyrebird-stream-manager.sh restart

Configuration-Only Backup

# Just the audio device configuration
sudo cp /etc/mediamtx/audio-devices.conf \
  /etc/mediamtx/audio-devices.conf.backup.$(date +%Y%m%d)

# Restore
sudo cp /etc/mediamtx/audio-devices.conf.backup.20250101 \
  /etc/mediamtx/audio-devices.conf

Debug Mode

Enable verbose output for all scripts:

# Enable debug for current session
export DEBUG=1

# Now all commands produce verbose output
sudo ./lyrebird-stream-manager.sh start
sudo ./usb-audio-mapper.sh --test
sudo ./lyrebird-diagnostics.sh full

Per-Script Debug Flags

# USB mapper debug
sudo ./usb-audio-mapper.sh -D

# MediaMTX installer verbose
sudo ./install_mediamtx.sh --verbose install

# Diagnostics debug
sudo ./lyrebird-diagnostics.sh debug

Complete Uninstallation

To fully remove LyreBirdAudio and all components:

# 1. Stop all streams and services
sudo ./lyrebird-stream-manager.sh stop
sudo systemctl stop mediamtx-audio 2>/dev/null
sudo systemctl disable mediamtx-audio 2>/dev/null

# 2. Uninstall MediaMTX
sudo ./install_mediamtx.sh uninstall

# 3. Remove systemd services and cron jobs
sudo rm -f /etc/systemd/system/mediamtx-audio.service
sudo systemctl daemon-reload
sudo rm -f /etc/cron.d/mediamtx-monitor

# 4. Remove USB device rules
sudo rm -f /etc/udev/rules.d/99-usb-soundcards.rules
sudo udevadm control --reload-rules
sudo udevadm trigger

# 5. Remove configuration files
sudo rm -rf /etc/mediamtx/
sudo rm -rf /etc/lyrebird/

# 6. Remove state and runtime files
sudo rm -f /run/mediamtx-audio.*
sudo rm -rf /var/lib/mediamtx-ffmpeg/
sudo rm -rf /var/lib/mediamtx/
sudo rm -rf /var/lib/lyrebird/

# 7. Remove log files
sudo rm -f /var/log/mediamtx.out
sudo rm -f /var/log/lyrebird-stream-manager.log
sudo rm -rf /var/log/lyrebird/
sudo rm -f /etc/logrotate.d/mediamtx

# 8. Remove the repository
cd ~
rm -rf /path/to/LyreBirdAudio

Verify Complete Removal

# Check for remaining processes
ps aux | grep -E "mediamtx|ffmpeg" | grep -v grep

# Check for remaining services
systemctl list-units | grep -E "mediamtx|lyrebird"

# Check for remaining config
ls /etc/mediamtx/ 2>/dev/null || echo "✓ Config removed"
ls /etc/udev/rules.d/99-usb-soundcards.rules 2>/dev/null || echo "✓ udev rules removed"
ls /var/lib/mediamtx-ffmpeg/ 2>/dev/null || echo "✓ State removed"

Partial Cleanup (Keep MediaMTX)

# Remove only streaming state (keep MediaMTX installed)
sudo ./lyrebird-stream-manager.sh stop
sudo rm -rf /var/lib/mediamtx-ffmpeg/
sudo rm -f /run/mediamtx-audio.*
sudo rm -f /var/log/lyrebird-stream-manager.log
