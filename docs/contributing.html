Getting Started

Prerequisites

# Install development dependencies (Ubuntu/Debian)
sudo apt install -y shellcheck bats

# macOS
brew install shellcheck bats-core

Fork and Clone

# 1. Fork on GitHub, then clone your fork
git clone https://github.com/YOUR_USERNAME/LyreBirdAudio.git
cd LyreBirdAudio

# 2. Add upstream remote
git remote add upstream https://github.com/tomtom215/LyreBirdAudio.git

# 3. Make scripts executable
chmod +x *.sh

Verify Development Setup

# Syntax check all scripts
for script in *.sh; do
    bash -n "$script" && echo "✓ $script"
done

# ShellCheck all scripts
shellcheck *.sh

# Run test suite (requires BATS)
cd tests && bats *.bats

Code Standards

Required Header Format

Every script must begin with a standard header:

#!/bin/bash
# script-name.sh - Brief Description
# Part of LyreBirdAudio - RTSP Audio Streaming Suite
# https://github.com/tomtom215/LyreBirdAudio
#
# Author: Your Name (https://github.com/yourusername)
# Copyright: Tom F and LyreBirdAudio contributors
# License: Apache 2.0
#
# Version: X.Y.Z
#
# DESCRIPTION:
#   Detailed description of what the script does.
#   Include any important behavioral notes.
#
# USAGE:
#   ./script-name.sh [options] command
#
# DEPENDENCIES:
#   - bash 4.0+
#   - ffmpeg, curl, etc.

set -euo pipefail

Safety Requirements

  • Always use set -euo pipefail at the top of every script
  • Always quote variables: "${variable}" not $variable
  • Use local for all function-scoped variables
  • Use readonly for constants
  • Validate all inputs at system boundaries (user input, external APIs)
  • Use mktemp for all temporary files
  • Implement signal handlers in long-running scripts: trap cleanup EXIT INT TERM

Style Guidelines

Naming Conventions

Formatting Rules

  • 4 spaces for indentation — no tabs
  • Maximum line length: 100 characters
  • Use [[ ]] instead of [ ] for conditionals
  • Use (( )) for arithmetic
  • Add comments for non-obvious logic

Logging Pattern

# Use the shared logging functions (from lyrebird-common.sh)
log_debug "Processing device: ${device_name}"
log_info "Stream started: ${stream_url}"
log_warn "Device busy, retrying in ${delay}s"
log_error "Failed to start MediaMTX: ${error_msg}"

Atomic File Operations

# Always write to temp first, then move atomically
local tmp_file
tmp_file=$(mktemp "${config_file}.tmp.XXXXXX")
trap 'rm -f "${tmp_file}"' EXIT

cat > "${tmp_file}" << EOF
# Configuration generated by lyrebird-mic-check.sh
${config_content}
EOF

mv "${tmp_file}" "${config_file}"

Testing

Running Tests

# Run all tests
bats tests/

# Run a specific test file
bats tests/test_lyrebird_common.bats

# Run tests matching a pattern
bats --filter "validation" tests/

# Verbose output
bats --verbose-run tests/

# TAP format (for CI)
bats --tap tests/

Writing Tests

#!/usr/bin/env bats

# Source the script being tested
setup() {
    # Load the script or helper
    source "${BATS_TEST_DIRNAME}/../lyrebird-common.sh" 2>/dev/null || true
}

@test "lyrebird_command_exists returns 0 for bash" {
    run lyrebird_command_exists bash
    [ "$status" -eq 0 ]
}

@test "log_info outputs formatted message" {
    run log_info "Test message"
    [ "$status" -eq 0 ]
    [[ "$output" =~ "Test message" ]]
}

Testing Requirements Before PR

  • ☐ All existing tests pass: bats tests/
  • ☐ New tests added for new functionality
  • ☐ bash -n script.sh passes (syntax check)
  • ☐ shellcheck script.sh passes with no errors
  • ☐ Manual testing on Ubuntu 20.04+ or equivalent
  • ☐ Tested with at least one real USB audio device (if applicable)

Contribution Workflow

  1. Create a feature branch 
    git checkout -b feature/your-improvement
# or
git checkout -b fix/issue-description
  2. Make your changes

    Follow the code standards and style guidelines above.

  3. Validate your changes 
    bash -n your-script.sh
shellcheck your-script.sh
bats tests/
  4. Test on target platforms

    Ubuntu 20.04+, Debian 11+, and Raspberry Pi OS if applicable. Test with actual USB devices.

  5. Submit a pull request
    • Clear description of changes and motivation
    • Reference any related issues (Closes #123)
    • Include test results
    • Update documentation if needed

Reporting Issues

Found a bug? Please open an issue on GitHub.

Include in Your Report

  1. System info: cat /etc/os-release && uname -a
  2. Script versions: From the Orchestrator menu or head -5 *.sh | grep Version
  3. Diagnostics: sudo ./lyrebird-diagnostics.sh full > diagnostics.txt 2>&1
  4. Relevant logs: Last 50 lines showing the error from /var/log/lyrebird-stream-manager.log
  5. Hardware info: lsusb && ./lyrebird-mic-check.sh

Feature Requests

Feature requests are welcome! Open a GitHub Discussion to discuss before opening a pull request for major features.

Code of Conduct

  • Be respectful and inclusive to all contributors
  • Focus on constructive, actionable feedback
  • Help others learn and grow
  • Keep discussions on-topic and technical
  • Assume good faith from other contributors
